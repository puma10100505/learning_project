# RecastNaviDemo：问题与解决过程记录

本文档整理在开发与调试 `RecastNaviDemo` 过程中遇到的典型问题、根因与处理方式，便于后续维护与排查。

---

## 1. MSVC 下 UTF-8 字符串与 ImGui

**现象**  
使用 `u8"..."`（C++20 `char8_t`）调用 `ImGui::Begin` 等以 `const char*` 为参数的 API 时，MSVC 报错，无法从 `const char8_t *` 转换到 `const char *`。

**处理**  
- 在 `RecastNaviDemo` 的 `CMakeLists.txt` 中对 MSVC 增加 `target_compile_options(..., /utf-8)`，使**普通**字符串按 UTF-8 解释。  
- UI 中移除 `u8` 前缀，使用 UTF-8 源文件 + `/utf-8` 的常规 `"中文"` 字面量。

---

## 2. ImGui 中文乱码、字体小且粗

**现象**  
编码正确后界面仍出现中文乱码；或合并 CJK 字体到拉丁字体后显示为小字、笔画过粗、可读性差。

**处理**（在 `DirectX12Window.cpp` 中）  
- 以 **CJK 字体为主字体**（`io.FontDefault`），而不是仅合并为副字体。  
- **优先使用 Windows 系统字体**（如 `C:/Windows/Fonts/msyh.ttc`，`ImFontConfig::FontNo = 0` 处理 TTC 集合），字号可调到约 18px。  
- 回退链：雅黑 / SimHei / 项目内打包 TTF 等。  
- **不要**在临时 `std::string` 上把 `.c_str()` 存进静态候选表 —— 会导致悬空指针。候选路径应存为 `std::string` 等持久对象，再 `c_str()` 传给 `AddFontFromFileTTF`。

---

## 3. 性能趋势窗口中 Clear 导致崩溃

**现象**  
点击「Clear」后程序崩溃。

**根因**  
在 `DrawBuildStatsWindow` 一类逻辑里，先用局部变量 `n` 表示历史长度，中途调用 `g_BuildHistory.clear()` 后仍用旧的 `n` 去画图表或遍历，造成越界或无效迭代。

**处理**  
历史长度与 `empty()` 在**每次**绘制/按钮响应后**重新读取**；`clear()` 之后**立即**以新长度进行后续循环，不缓存过期的大小。

---

## 4. Windows 头文件与 `small` 宏冲突

**现象**  
某处使用局部变量名 `small`（如 `ImVec2 small`），与 Windows RPC 头中的 `typedef short small` 冲突，编译失败。

**处理**  
重命名该变量（例如为 `miniSz`）。

---

## 5. 底部性能面板布局与“Dock”行为

**现象**（需求演进）  
ImGui 1.86 无原生 Docking 时，需要主窗口为底部统计区让出高度、可拖拽分隔条等；后续又要求去掉 “Dock to Bottom” 勾选项、固定为底部等。

**处理**  
- 用全局状态（如 `g_ShowStatsWindow`、`g_StatsDockHeight`）配合 `SetNextWindowPos/Size` 和主窗口 `WorkSize` 缩短，模拟底部停靠。  
- 自定义 `HSplitterBottom` / `VSplitter` 等，用 `InvisibleButton` + `MouseDelta` 调高度或左右宽度。  
- 多系列趋势图用 `ImDrawList` 自绘 + 可点击图例，替代当时工程内未启用的 ImPlot 路径（若项目有 ImPlot 可再考虑迁移）。

---

## 6. OBJ 加载后场景中“无显示 / 无着色”

### 6.1 无显示

**可能原因**  
- 3D 相机 `Target`、距离仍针对旧 procedural 场景，OBJ 包围盒差异大时整模型在视锥外或“埋在”相机里。  
- 画布**从未绘制**输入几何，仅画障碍、NavMesh 多边形等；OBJ 无障碍时且 NavMesh 未建好时右侧看起来为空。

**处理**  
- 提供 `FrameCameraToBounds()`：根据 `g_Geom.Bounds` 设置 `Target`、估计 `Distance` 与合理 `Yaw`/`Pitch`。在 OBJ 加载成功、`Use Procedural`、Scene 尺寸变化等场景调用。  
- 在 `DrawCanvas2D` / `DrawCanvas3D` 中增加**输入三角网格的线框**（主要面向 OBJ 导入时的可视反馈；见下节“双 Grid”的取舍）。  
- 透视远裁剪、相机最大距离在超大模型上适当加大（如 far plane 5000、max distance 800 等，以实际调参为准）。

### 6.2 有网格线但 NavMesh 无绿色着色

**原因**  
曾对 `BuildNavMesh` 使用 `rcMarkWalkableTriangles` **完全重算** `triareas`，忽略工程里为 procedural 已写入的 `RC_NULL_AREA`（障碍下方不可走），或掩盖 OBJ 上本应按斜率剔除的面。

**处理**  
- 以 `g_Geom.AreaTypes` 为初值，再调用 **`rcClearUnwalkableTriangles`**（按 `walkableSlopeAngle` 清除陡坡为 `RC_NULL_AREA`），**不会**把已标记为 `RC_NULL_AREA` 的障碍格恢复为可行走。  
- procedural：障碍下三角形仍用 `RC_NULL_AREA` 参与构建。  
- OBJ：初始全 `RC_WALKABLE_AREA` 时，由 `rcClearUnwalkableTriangles` 处理墙面、陡坡等。

### 6.3 Procedural 下“双重 Grid”

**原因**  
在 procedural 中“地面”本身就是稠密三角网格，再画一层**输入几何线框**会显得像两套网格重叠。

**处理**  
输入线框**仅**在 `GeomSource::ObjFile` 时绘制；procedural 保留原有显式 `Grid` 与障碍绘制。

## 7. 使用“打开文件”选择 OBJ，而非手输路径

**处理**  
在 Windows 下使用 `GetOpenFileNameA` / `GetSaveFileNameA`（`commdlg.h`，链接 `comdlg32`），`OFN_NOCHANGEDIR` 避免改进程当前工作目录。NavMesh 的保存/加载同样可用对话框。父窗口 `GetActiveWindow()`。

---

## 8. OBJ 下起点/终点在逻辑上正确，但胶囊体位置错误

**根因**  
`DrawCapsule3D` 在内部把胶囊**底部**锚在 **Y=0**，而 OBJ 上合法站立高度常为 `g_Start[1] / g_End[1] != 0`。

**处理**  
为 `DrawCapsule3D` 增加 `groundY`（或等价参数），所有环与半球、圆柱段在**世界高度**上整体平移到 `groundY`；调用处传入 `g_Start[1]`、`g_End[1]`。

---

## 9. 起点/终点点击后无法贴到有效 NavMesh（OBJ）

**根因**  
- 3D 拣取只与 **y=0 平面**求交，而网格在 y≠0。  
- `g_Start[1]` 等未更新，Detour `findNearestPoly` 的 `halfExtents[1]` 过小，垂直方向搜不到。  

**处理**  
- 对 OBJ 做视射线与**输入三角形**的 Möller–Trumbore 求交，得到命中点 `(x, y, z)`；procedural 可退回与 `bounds` 中间高度平面求交。  
- 2D 顶视图无真实 Y 时，可用包围盒中心 Y 作兜底。  
- `findNearestPoly` 的 Y 向半长按场景高度自适应放大。  
- 加载 OBJ 后可将起终点**初始化到包围盒内**合理位置，避免仍停留在旧 (-10,0) 等坐标。

---

## 10. Wasd 与快捷键冲突、相机平移

**需求**  
WASD 用于场景内平移；若与旧快捷键冲突，**优先** Wasd。

**处理**  
- 将原 **S = Select & Move** 改为 **T**。  
- 在 `Orbit3D` 下根据 `Yaw` 在 XZ 平面求前/右向量，按 `DeltaTime` 与 `Distance` 缩放平移 `g_Cam.Target`；Q/E 调整 `Target.y`；Shift/Ctrl 加速/减速。  
- **关键**：`DrawCanvas3D` 中**不要**每帧把 `g_Cam.Target` 强行重置为包围盒中心，否则平移下一帧被抹除；`Target` 只由 `FrameCameraToBounds` / 用户操作更新。  
- Scene 半尺寸等变更后调用 `FrameCameraToBounds()`，与手动平移的语义区分清楚。

---

## 11. 重置相机（Reset Camera）

**处理**  
- View 面板中按钮与 **`Home` 键** 调用与「拉到包围盒」相同逻辑 `FrameCameraToBounds()`，命名上体现「重置到当前几何包围盒的合理视角」。

---

## 12. 构建成功但 post-build 复制 exe 失败

**现象**  
`cmake --build` 在链接成功后报错：无法将 `RecastNaviDemo.exe` 复制到 `Bin/` 等目标目录。

**根因**  
目标路径上的 exe 仍被**正在运行的进程**占用，Windows 下无法覆盖。

**处理**  
关闭正在运行的 `RecastNaviDemo`（或从 `Bin` 外路径运行并结束旧进程）后重新 build；或直接使用 `cmake-build-debug/.../Debug/RecastNaviDemo.exe` 调试。
